How Auto-Generation Works

When you click Generate on the Play Periods page:

  1. The system finds the latest end date in the DB (or calculates from your start date).
  2. The next night starts on the first Monday on or after that date.
    3. {% if settings.has_midnight and settings.xp_frequency == 'weekly' %}
  3. Because Midnight + Weekly is on, each night creates two periods:
    • Dusk to Midnight — Mon → Sat (6 days)
    • Midnight to Sunrise — Sat → Sat (8 days)
    4. {% else %}
  4. Each night runs for {{ settings.night_duration_days }} days.
    5. {% endif %}
  5. After every {{ nights_per_cycle }} night{{ 's' if nights_per_cycle != 1 }} (~{{ settings.timeskip_interval_weeks }} weeks), a {{ settings.downtime_duration_days }}-day downtime is inserted.
  6. Periods that already exist are skipped — safe to run multiple times.

Night numbering: Downtime entries get night number 0. Only play nights increment the counter. The counter is calculated from your server start date when the DB is empty.

If you delete all periods and regenerate, the generator starts from the current ongoing night — it won't backfill historical nights. Create those manually if needed.
