How Auto-Generation Works

When you click Generate on the Play Periods page:

  1. The system finds the latest end date in the DB.
  2. The next night starts on the first Monday after that date.
  3. It runs for {{ settings.night_duration_days }} days.
  4. After every {{ nights_per_cycle }} night{{ 's' if nights_per_cycle != 1 }} (~{{ settings.timeskip_interval_weeks }} weeks), a {{ settings.downtime_duration_days }}-day downtime is inserted before the next night.
  5. Nights that already exist are skipped — safe to run multiple times.

Night numbering: Downtime/timeskip entries get night number 0. Only play nights increment the counter. NYbN is currently on Night {{ settings.server_start_date | replace('-', '') }} — if the DB is empty, generation starts at Night 62.
